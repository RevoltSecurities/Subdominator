# 🚀 v3.0.0 - The Precision Discovery Release

We are excited to announce the release of Subdominator v3.0.0. This milestone represents a complete evolution of the engine, focusing on massive-scale discovery, deep asset mapping, and a refined professional experience.

### ➕ Added
- **Interactive Auditing Shell**: A specialized terminal for searching, filtering, and auditing massive subdomain datasets without leaving the CLI.
- **Diagnostic Health Engine**: Integrated connectivity and API status verification to ensure your reconnaissance environment is production-ready.
- **Global Container Registry**: Official Docker images are now published directly to the GitHub Container Registry (`ghcr.io`).
- **Automated CI/CD**: Seamless release pipelines for both PyPI distribution and Docker delivery.

### ✨ Enhanced
- **High-Capacity Discovery**: Optimized findings cache for scanning 100K+ subdomains with peak stability and minimal memory footprint.
- **Deep Reconnaissance Mapping**: Automated recursive enumeration allows you to uncover hidden attack surfaces with configurable depth and precision.
- **73+ OSINT Integrations**: Support for an expansive catalog of 70+ resources, significantly increasing the success rate for obscure targets.
- **Professional Workflow**: Standardized on the `uv` ecosystem for near-instant installations and predictable development environments.
- **Stakeholder Evidence**: Rebuilt HTML reporting engine for cleaner, more impactful presentation of security findings.

### 🔨 Fixed
- **Slimmed Footprint**: Removed heavyweight dependencies and eliminated system-level `Fontconfig` warnings, significantly reducing package weight.
- **Stability Core**: Resolved platform-specific lazy loading and connection pooling logic for `asyncpg` and `aiohttp`.
- **Resource Management**: Fixed memory spikes during high-volume enumeration runs through optimized disk-backed persistence.
